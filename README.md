# Responsible Agentic Development Framework (RADF)

> **HW1 skeleton.** Sections marked _TBD_ are filled in during Phase 3 (`TODO.md` T3.3).

A framework that keeps a persistent knowledge graph of a codebase — components, dependencies,
and the decisions behind them — so that agent sessions build on accumulated architectural
knowledge instead of re-deriving it by grepping the repo every time.

**HW1 delivers:** a single raw-Python agent over four knowledge-graph tools, with an
observe → reason → act → verify loop, explicit stopping conditions, a human approval gate on
the one irreversible action, and a tool-error branch. No frameworks.

## Tools

| Tool | Effect | Constrained param | Reversible | Gated |
|---|---|---|---|---|
| `scan_repository_structure` | writes nodes/edges to the graph | `kind` enum, `max_depth` int | yes (re-runnable) | no |
| `query_component_graph` | read-only lookup | `relation` enum | yes | no |
| `append_decision_record` | appends to decision log | `status` enum | yes (append-only) | no |
| `verify_graph_integrity` | domain check, returns structured error | `scope` enum | yes | no |
| `prune_graph_node` | permanently deletes a node | `cascade` enum | **no** | **yes** |

---

## Safety properties

- **Stopping conditions:** `answered` · `max_steps` · `stalled` (repeated identical call) ·
  `declined`. Stopping is the code's decision, not the model's.
- **Approval gate:** `prune_graph_node` is the only irreversible action and the only gated
  one. It proceeds on explicit confirmation; a decline returns a `declined` result to the
  model rather than silently failing.
- **Error branch:** `verify_graph_integrity` returns a structured error that the loop routes
  to its own branch — a corrupt graph never re-enters the model's context as valid data.
- **Untrusted tool output:** text returned by a tool is treated as data, never as instructions.

## Constraints (HW1)

Standard library, `openai`, `python-dotenv`, `pytest`. No agent/LLM frameworks, no vector or
graph databases, and no external code indexers — structural extraction is hand-rolled with `ast`
this round, on purpose. See [`CLAUDE.md`](CLAUDE.md) §4. These open up in later homeworks on this
same repository.

---

## Design principle: structure is derived, decisions are authored

Nodes and edges are regenerated by any scan and never hand-edited. Decision records are authored,
durable, and never overwritten by a scan. The two are joined by `symbol_uid` — a decision
*references* a component, it is never stored *inside* one. A decision whose uid stops resolving is
orphaned and surfaced for review, not dropped.

This keeps the decision layer portable: when structural extraction is later delegated to an
external indexer, only the derived half is replaced. See [`CLAUDE.md`](CLAUDE.md) §6 and
[`ARCHITECTURE.md`](ARCHITECTURE.md) §6.1.
